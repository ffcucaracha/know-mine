# Инструкция пользователя Know Mine

Know Mine помогает задавать вопросы по загруженным научно-техническим документам, смотреть источники ответа, сохранять полезные фрагменты в избранное и изучать карту знаний.

## 1. Запуск

Запустите приложение:

```bash
streamlit run app.py
```

Откройте адрес, который Streamlit покажет в терминале. Обычно это:

```text
http://localhost:8501
```

## 2. Основные вкладки

- `Вопрос` — задать вопрос по базе источников и получить ответ.
- `Избранное` — посмотреть сохранённые источники и фрагменты.
- `История отчётов` — открыть и скачать ранее сформированные Markdown-отчёты.
- `Карта знаний` — найти сущность и посмотреть связи.
- `Администрирование` — загрузка документов, индексация, извлечение фактов и состояние системы.

## 3. Как задать вопрос

1. Откройте вкладку `Вопрос`.
2. Выберите пример вопроса или введите свой.
3. Настройте `Количество фрагментов`, если нужно больше или меньше источников.
4. Нажмите `Ответить`.

После ответа на странице появятся:

- итоговый ответ;
- `Источники`;
- `Точные фрагменты из поиска`;
- `Факты из графа`;
- кнопка скачивания Markdown-отчёта.

## 4. Источники ответа

Блок `Источники` показывает документы, на которых основан ответ.

Каждый источник можно раскрыть. Внутри отображается:

- путь к документу;
- страницы;
- найденный фрагмент;
- предпросмотр документа;
- кнопка `Добавить в избранное`;
- кнопка `Открыть документ`, если файл доступен локально.

Если добавить источник в избранное, раскрытый источник остаётся открытым, а внутри появится сообщение об успешном добавлении.

## 5. Избранное

Во вкладке `Избранное` можно:

- посмотреть сохранённые источники;
- искать по имени файла или фрагменту;
- удалить отдельный источник;
- очистить весь список избранного.

Избранное хранится локально в SQLite, поэтому сохраняется после перезапуска приложения.

## 6. История отчётов

Каждый успешный ответ автоматически сохраняется как Markdown-отчёт.

Во вкладке `История отчётов` можно:

- найти отчёт по вопросу;
- посмотреть preview;
- скачать Markdown повторно;
- удалить один отчёт;
- очистить всю историю.

## 7. Карта знаний

Во вкладке `Карта знаний` можно искать сущности из базы: материалы, процессы, оборудование, публикации, страны и другие типы.

Как пользоваться:

1. Выберите пример сущности или введите свою.
2. Нажмите `Показать связи`.
3. Выберите найденный узел.
4. Посмотрите граф связей.

Под картой есть подсказка по управлению: узлы можно перетаскивать, масштаб меняется колесом мыши, при наведении показываются подробности.

## 8. Загрузка документов

Загрузка находится во вкладке `Администрирование` -> `Загрузка`.

Есть два режима:

- `Маленький архив через браузер` — для небольших ZIP-архивов.
- `Большой архив / папка по локальному пути` — для больших папок и архивов, особенно 1+ ГБ.

Для больших наборов документов лучше использовать локальный путь:

```env
SOURCE_PATH=data/raw/sources
```

Обычный порядок обработки:

1. Сканировать путь или распаковать архив.
2. Нажать `Использовать эти файлы`.
3. Нажать `Извлечь текст`.
4. Нажать `Создать чанки`.
5. Нажать `Построить векторный индекс`.
6. При необходимости открыть вкладку `Факты` и запустить извлечение фактов и связей.

## 9. Ограничения демо-режима

Количество обрабатываемых документов и чанков ограничивается настройками:

```env
DEMO_MAX_DOCUMENTS=5
DEMO_MAX_CHUNKS=30
```

Если документов больше лимита, приложение обработает только первые документы в пределах лимита.

## 10. Если ответ выглядит неполным

Проверьте:

- загружены ли документы;
- выполнено ли извлечение текста;
- созданы ли чанки;
- построен ли векторный индекс;
- хватает ли значения `Количество фрагментов` во вкладке `Вопрос`;
- есть ли релевантные документы по теме вопроса.

Для вопросов на сравнение полезно увеличить количество фрагментов, чтобы в контекст попали документы по обеим сравниваемым сущностям.
